# NPX 介绍
`npx`之所以能直接运行命令，核心原因在于它是 npm（Node Package Manager）官方自带的一个**包执行器（Package Executor）**。它的设计初衷就是为了解决“临时需要用一个工具，但又不想每次都去全局安装它”的痛点。

它能直接运行命令，背后有以下三个核心工作机制：

### 1. 自动查找本地 `node_modules`

当你输入 `npx <command>` 时，它首先会像雷达一样扫描你当前项目目录：

1. 检查当前目录下的 `node_modules/.bin/` 文件夹，看看里面有没有你想运行的命令。
2. 如果你在当前项目里用 `npm install tsx` 安装过，那么直接运行 `npx tsx` 就会秒级启动本地的这个工具。

> **好处：** 你不需要在 `package.json` 的 `scripts` 里写一长串配置，也能直接调用本地安装的依赖。

### 2. “即用即焚”的远程下载机制

如果本地项目和全局环境都**没有**这个命令（比如你第一次使用 `npx create-react-app my-app`），`npx` 会执行以下骚操作：

1. 默默把最新版的工具下载到一个内部的**临时缓存目录**。
2. 在临时目录里直接执行该命令。
3. 执行完毕后，它不会把这个包留在你的电脑里，而是随后在合适的时候自动清理。

> **好处：** 你的电脑不会因为尝试各种新工具而堆满几百兆的全局垃圾（Global Pollution），同时保证了你每次临时调用用的都是**最新版本**。

### 3. 省去了配置环境变量（PATH）的痛苦

在没有 `npx` 之前，如果你把一个工具安装在本地（没有加 `-g`），你想在终端运行它，你必须写出它的完整路径：

```bash
./node_modules/.bin/tsx ./type/primitive_type_01.ts

```

而 `npx` 相当于一个快捷方式，它会自动把 `./node_modules/.bin` 临时加入到系统的环境变量中，让你只需要输入 `npx tsx`，它就能自己对齐路径。

---

### 一句话总结

| 方式 | 相当于 | 结果 |
| --- | --- | --- |
| **`npm install -g`** | 把工具**买回家**放在客厅 | 占地方，久了不更新变老旧 |
| **`npx`** | 去租车公司**租一辆车**，用完就还 | 不占车位，每次都能开到最新的车型 |
